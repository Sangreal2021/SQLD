1. 카디널리티(Cardinality) : 하나의 릴레이션(테이블)에서 튜플(행)의 전체 개수

2. 튜플 : 행(ROW)이라고도 하며, 하나의 테이블에 저장되는 값

3. 릴레이션 : 데이터베이스의 테이블을 의미.
	- 정형화된 구조: 릴레이션은 열(속성)과 행(튜플)으로 구성되며, 각 열은 고유한 이름과 데이터 타입을 가짐
	- 고유한 키: 각 릴레이션에는 튜플을 유일하게 식별할 수 있는 하나 이상의 열로 구성된 키(주 키)가 있음
	- 무순서성: 릴레이션 내의 튜플은 순서가 없으며, 데이터를 검색할 때 특정 순서로 정렬해야 하는 경우 SQL 쿼리에서 명시적으로 지정해야 함

4. 선택도 : 특정 조건에 의해서 선택될 것으로 예상되는 레코드의 비율

5. BCNF(Boyce-Codd Normal Form) : 기본키 제외, 후보키가 존재하고 후보키가 기본키의 속성 중 하나를
	함수적으로 종속하는 경우 발생.
